lametric-hass-local

Local Home Assistant integration for LaMetric devices using the device API (lametric-py) with Zeroconf / SSDP / DHCP discovery and a config flow.

Home Assistant HACS local polling MIT

Entities

Button, Light (SKY), Number, Scene, Select, Sensor, Switch, Text, and Update.

Services

show_message, show_chart, and pixel streaming (SKY). Screensaver control via entity service.

Discovery

Automatic device detection via Zeroconf, SSDP, and DHCP.

Config Flow

Manual entry (IP + API key) or cloud-assisted OAuth setup.

Installation

The recommended way is to install via HACS:

  1. Open HACS in Home Assistant and go to Integrations.
  2. Click the three-dot menu → Custom repositories.
  3. Add ElectroAttacks/lametric-hass-local with category Integration.
  4. Search for LaMetric Local and install it.
  5. Restart Home Assistant.

Alternatively, copy custom_components/lametric_hass_local/ manually into your HA custom_components/ folder and restart.

Configuration

Go to Settings → Devices & Services → Add Integration and search for LaMetric Local.

Manual setup

Enter the device IP address and the API key shown under Settings → Wi-Fi and Settings → Developer → API Key in the LaMetric app.

Cloud-assisted OAuth

You can let the integration fetch available devices from developer.lametric.com using OAuth. A developer account is required. The device is still controlled locally after setup.

Discovery via Zeroconf, SSDP, or DHCP will pre-fill the IP for you. Just confirm and enter the API key.

Service: show_message

Sends a text notification with an optional icon and sound to a LaMetric device.

Field Required Default Description
device_id yes — Target LaMetric device
message yes — Text to display
icon no — Icon ID from developer.lametric.com/icons
sound no — Sound ID (e.g. positive1, alarm3)
cycles no 1 How many times to repeat the notification (0 = infinite)
icon_type no none none / info / alert
priority no info info / warning / critical 
service: lametric_hass_local.show_message
data:
  device_id: "your-device-id"
  message: "Deploy finished"
  icon: "a7956"
  sound: "positive1"
  cycles: 1
  priority: info

You can also use the legacy notify.lametric_* platform service with the same optional fields passed under data:.

Service: show_chart

Sends a spike chart notification built from a list of numeric values.

Field Required Default Description
device_id yes — Target LaMetric device
data yes — List of numbers to plot, e.g. [1,2,3,4,5,4,3,2,1]
sound no — Sound ID
cycles no 1 Repeat count
icon_type no none none / info / alert
priority no info info / warning / critical 
service: lametric_hass_local.show_chart
data:
  device_id: "your-device-id"
  data: [1, 3, 5, 7, 5, 3, 1]
  sound: "statistic"

Entity Service: set_screensaver

Enables or disables the screensaver and optionally configures its mode. This is an entity service — target the brightness_mode select entity of the device.

mode_params is only evaluated when mode is set to time_based.
Field Required Default Description
enabled yes — Turn screensaver on or off
mode no when_dark when_dark / time_based
mode_params no — Only used with time_based: enabled, start_time, end_time 
service: lametric_hass_local.set_screensaver
target:
  entity_id: select.lametric_brightness_mode
data:
  enabled: true
  mode: time_based
  mode_params:
    enabled: true
    start_time: "2000-01-01T22:00:00"
    end_time: "2000-01-01T08:00:00"

Pixel Streaming (SKY)

LaMetric SKY supports the LaMetric Streaming Protocol (LMSP) for pushing raw RGB888 frames over UDP. The flow is:

  1. Start a stream — start_stream — returns a session_id.
  2. Send frames in a loop — send_stream_data.
  3. Stop the stream — stop_stream.
Pixel streaming is only available on LaMetric SKY devices. The service calls will return an error on other models.

Entity Service: start_stream

Starts an LMSP stream session on the SKY light entity. Returns a session_id that must be passed to send_stream_data. 

service: lametric_hass_local.start_stream
target:
  entity_id: light.lametric_sky_light
data:
  config:
    canvas:
      fill_type: scale        # scale | tile
      render_mode: pixel      # pixel | triangle
      post_process:
        type: none            # none | effect

With fading pixels effect

service: lametric_hass_local.start_stream
target:
  entity_id: light.lametric_sky_light
data:
  config:
    canvas:
      fill_type: scale
      render_mode: pixel
      post_process:
        type: effect
        params:
          effect_type: fading_pixels
          effect_params:
            smooth: true
            pixel_fill: 1.0
            fade_speed: 0.005
            pixel_base: 0.05

Entity Service: send_stream_data

Sends one RGB888 frame to an active LMSP stream session.

Field Required Description
session_id yes Session ID returned by start_stream
rgb_data yes List of [R, G, B] triplets, one per pixel, left-to-right top-to-bottom 
service: lametric_hass_local.send_stream_data
target:
  entity_id: light.lametric_sky_light
data:
  session_id: "{{ session_id }}"
  rgb_data: "[[255,0,0],[255,0,0],[0,255,0],...]"

Use stop_stream to end the session:

service: lametric_hass_local.stop_stream
target:
  entity_id: light.lametric_sky_light

Scenes

Each installed LaMetric app widget is exposed as a Home Assistant scene entity. Activating a scene brings the corresponding widget to the foreground. The is_active attribute reflects whether the widget is currently visible on the device.

Entity Service: activate_action

To trigger a specific widget action, use the activate_action entity service instead of scene.turn_on:

Field Required Default Description
action_id yes — ID of the widget action to trigger
action_parameters no — Key/value map of parameters required by the action
visible no true Bring the widget to the foreground after triggering 
service: lametric_hass_local.activate_action
target:
  entity_id: scene.lametric_weather
data:
  action_id: "forecast"
  action_parameters:
    location: "Berlin"
  visible: true
Calling scene.turn_on activates the widget without triggering any action.

All Entities

Platform Description
Button Navigate to the next or previous app, dismiss the current notification, or dismiss all notifications
Light Controls the SKY display brightness and on/off state (SKY only)
Number Sets display brightness (non-SKY, %) and audio volume (%); ranges are reported by the device
Scene One entity per installed app widget — activates the widget
Select Controls the brightness mode (auto / manual) of the display (non-SKY)
Sensor Wi-Fi signal strength (%) — diagnostic, disabled by default
Switch Toggles display on/off and Bluetooth on/off
Text Sets the Bluetooth display name of the device
Update Tracks installed and latest firmware version; available on OS 2.3.0 and later
